## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser ← User input (URL search params) | `?legacy_nav=1` flag is read from `window.location.search` and used to switch render path. Untrusted input crosses here. |
| Browser ← User input (touch events) | swipe-left dismissal reads touchstart/touchend coordinates. |
| Browser ← User input (search query) | SearchOverlay input forwards user text to existing search infrastructure. |
| Browser ← /api/daemon-status response | DaemonStatusIndicator fetches daemon-status JSON; same endpoint already used by SocialFeed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-05 | Tampering | `?legacy_nav=1` query flag | accept | Read-only render switch with no privileged operation gated. Worst case: a user sees the legacy header. No data exposure, no auth bypass, no write path. |
| T-10-06 | Spoofing | hamburger aria-expanded mismatch | mitigate | Bind `aria-expanded` directly to `drawerOpen` state; never set manually. Playwright a11y test in Phase F asserts the binding. |
| T-10-07 | Information Disclosure | SearchOverlay query input | accept | Input forwards to existing `SearchBar` query path; no new endpoint. Same trust surface as today's search. |
| T-10-08 | DoS | swipe-left handler firing on every touchmove | mitigate | Use touchstart + touchend only (no touchmove). No throttling needed because touchend fires once per gesture. |
| T-10-09 | Tampering | `history.pushState` on overlay open | accept | History entry has no sensitive data (`{ searchOverlayOpen: true }` only). Popstate listener verifies state shape before acting. |
| T-10-09b | Information Disclosure | DaemonStatusIndicator /api/daemon-status | accept | Same endpoint SocialFeed already calls; refactor only moves the call site, no new exposure. |
