<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser ← CDN (fonts) | Fonts previously loaded from `fonts.googleapis.com`; now self-hosted via next/font. Reduces third-party trust surface. |
| Browser ← Vercel (static CSS) | `globals-terminal.css` + tailwind-generated CSS served from Vercel edge. No untrusted input injected. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | globals-terminal.css `@import "tailwindcss"` | accept | Static import resolved at build time by `@tailwindcss/postcss`; no runtime tampering vector. |
| T-10-02 | Information Disclosure | next/font self-hosted bundles | mitigate | Self-hosting eliminates referrer leak to Google Fonts CDN; confirms D-25. |
| T-10-03 | Denial of Service | prefers-reduced-motion global block | accept | `animation: none !important` is defensive, not a DoS vector; a malicious stylesheet could not override it because it lives in first-party CSS. |
| T-10-04 | Tampering | tailwind.config.ts content globs | accept | Build-time config; not user-reachable. Any local tampering requires filesystem access equivalent to source-code access. |
</threat_model>
