# Plan: Corruption Scorer v5

## Context

Investigation of Amy Acton (oh-gov-2026-amy-acton) surfaced two issues with her current DB record:

1. **Contribution breakdown is wrong.** DB shows 100% individual; reality is ~95% individual + ~5% PAC (Ohio Democratic Party $125K, Ohio Federation of Teachers PAC, Ohio State UAW PAC).
2. **Scoring algorithm has no signal for donor-pattern forensics.** A grassroots-heavy campaign with no itemized donor list is indistinguishable from a laundering operation on the current scorer. User raised a concern about this ‚Äî a content-neutral factor can capture it without ethno-religious profiling.

## Tasks

### 1. Add `donorForensicsScore` factor to `lib/corruption-score.ts`

- Rebalance `BASE_WEIGHTS` to sum 1.0:
  - `pacContributionRatio`: 0.35 ‚Üí **0.32**
  - `lobbyingConnections`: 0.20 ‚Üí **0.18**
  - `votingAlignment`: 0.25 ‚Üí **0.22**
  - `campaignFinanceRedFlags`: 0.20 ‚Üí **0.18**
  - `donorForensicsScore`: **0.10** (new)
- Add `scoreDonorForensics(p: Politician): CorruptionFactor` reading from `p.donorForensics` (new optional field on Politician type).
- Signals evaluated when data present:
  - `missing_employer_ratio` ‚Äî % itemized >$200 donors with blank/"Information Requested" employer
  - `geographic_anomaly` ‚Äî out-of-state % vs. expected baseline for office type
  - `household_bundling` ‚Äî max-contribution donors sharing an address
  - `donation_std_dev` ‚Äî abnormally uniform amounts (laundering signal)
  - `platform_opacity` ‚Äî heavy ActBlue/WinRed with no direct-committee disclosure
- Default `dataAvailable=false` (placeholder) until itemized data flows in. Placeholder weight redistribution is already handled by `getAdjustedWeights`.

### 2. Extend `Politician` type in `lib/types.ts`

Add optional `donorForensics?: { missingEmployerRatio: number; outOfStatePct: number; householdBundling: number; donationStdDev: number; platformOpacity: number; itemizedCount: number; computedAt: string; }` ‚Äî drives the new factor.

### 3. Extend `scripts/sync-oh-state-finance.ts`

- Emit it on `OhFinanceResult` when itemized data is present; skip (undefined) otherwise.
- Factor stays in placeholder mode across all politicians until OH SOS portal is reachable again.
